## 7. GitHub Projects를 활용한 프로젝트 관리 실습 (개인 저장소 기반)

이 실습은 기존의 이슈를 통합 관리하고, 다양한 '뷰'를 통해 프로젝트의 큰 그림을 파악할 수 있는 **GitHub Projects**의 기본 사용법을 익히는 것을 목표로 합니다.

**사용 저장소:** 모듈 2에서 만들었던 `github-practice-repo`를 계속 사용합니다.

---

### 4.  뷰 전환 (Table -> Board)

**데이터는 하나**이지만, '바라보는 관점'을 칸반(Kanban) 스타일의 보드 뷰로 변경합니다.

1.  현재 뷰 이름 (기본값: 'Table') 옆의 **'+'** 아이콘을 클릭하여 **'New View'**를 선택합니다.
2.  **View name**을 **`Status Board`**로 입력합니다.
3.  **Layout**을 **'Board'**로 선택합니다.
4.  **'Board by'** 필드는 기본값인 **`Status`** 필드를 사용합니다.
5.  **'Create'**를 클릭하면 보드 뷰가 생성됩니다.
    * 
6.  **드래그 앤 드롭:** 이슈 #1 카드를 마우스로 끌어 **'In Progress'** 열로 이동시킵니다.
7.  **실시간 반영 확인:** 다시 'Table' 뷰로 돌아가서 이슈 #1의 `Status` 필드가 자동으로 **'In Progress'**로 변경되었는지 확인합니다.

### 5. 📊 인사이트(Insights) 확인 (프로젝트 진행률 파악)

프로젝트 데이터가 쌓이면 GitHub가 자동으로 분석하여 차트를 제공하는 기능을 확인합니다.

1.  Projects 페이지 상단 탭에서 **'Insights'**를 클릭합니다.
2.  **'Charts'** 섹션에서 여러 차트 유형(예: **Item status**, **Burn down**)이 자동으로 생성된 것을 확인합니다.
3.  **'Item status'** 차트에서 현재 'To do'와 'In Progress' 상태의 항목이 몇 개인지 시각적으로 파악합니다.
    * **[핵심]** 이 차트들은 테이블 뷰의 데이터를 기반으로 **자동 생성**되며, PM이 프로젝트 진행 상황을 한눈에 파악하는 데 필수적인 도구입니다.

---

### 6. 🗑️ 프로젝트 항목 정리 (선택 사항)

실습을 완료했다면, Projects의 항목들을 정리합니다.

1.  'Status Board' 뷰에서 이슈 #2와 '문서화 검토'를 **'Done'**으로 옮겨 작업을 완료합니다.
2.  프로젝트의 모든 항목이 'Done'으로 표시되는 것을 확인합니다.
